A task is a conversation where you ask a Worker to get something done. Each task keeps its full history, the files it produces, and the result, so you can return to it anytime.

## Start a task

Open **New Task**, then describe what you want in the task box. Clear goals produce better results, so include the inputs, the outcome you expect, and the format you want back.

<Frame caption="Describe the task you want your Worker to complete">
  <img src="/images/dashboard-en.png" alt="New task input box" />
</Frame>

Attach files or paste text to give the Worker something to work on, such as a contract to review or data to analyze.

## Choose a mode

Modes tune the Worker for a type of legal work. Pick the one that matches your task:

| Mode | Use it for |
| --- | --- |
| Corporate & Commercial | Company and commercial matters |
| Intellectual Property | IP analysis and infringement checks |
| Distressed Assets | Distressed asset review |
| Compliance Review | Checking against rules and playbooks |
| Litigation Support | Case preparation and supporting analysis |
| Contract Drafting | Generating and editing contracts |
| Legal Presentation | Building slides and presentations |
| Data Analysis | Analyzing structured data and producing charts |

## Choose how the Worker runs

## Follow the work

While a task runs, you can watch each step, see the tools the Worker uses (such as web search or a virtual computer), and read results as they appear. Subtasks, like a contract review or an infringement comparison, are shown as they run.

</Frame>

## Refine and continue

Reply in the same task to ask for changes or follow-up work. The Worker keeps the earlier context, so you can iterate without starting over. Files the Worker generates are collected in [Space](/en/space).
