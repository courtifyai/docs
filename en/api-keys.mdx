API keys let you run tasks from your own scripts and applications instead of the web app. Create a key, then use it to start a task, check its status, get the result, and send follow-ups.

<Frame caption="API keys">
  <img src="/images/api-keys-en.png" alt="API keys screen" />
</Frame>

## Create a key

<Steps>
  <Step title="Open API keys">
    Go to **Settings** and open **API Keys**.
  </Step>
  <Step title="Create and copy">
    Create a new key and copy it immediately. Store it somewhere safe, since you may not be able to view it again.
  </Step>
</Steps>

## Manage keys

- **Revoke** — disable a key so it can no longer be used.
- **Delete** — remove a key permanently.

A key's status shows as active, **Revoked**, or **Expired**.

## Usage examples

The API keys screen includes copy-ready examples for common operations:

- Run a task
- Check a task's status
- Get a task's result
- Send a follow-up

<Warning>
  Treat API keys like passwords. Do not commit them to source control or share them. Revoke a key immediately if it may have been exposed.
</Warning>
